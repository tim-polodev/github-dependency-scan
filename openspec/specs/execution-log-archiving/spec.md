# Capability: Execution Log Archiving

## Purpose
Automatically captures and archives scanner execution logs into MongoDB at the completion of each run for persistent auditing and debugging.

## Requirements

### Requirement: Execution Log Buffering
The system SHALL capture and format all execution logs (stdout/stderr generated via the main logger and subprocesses) in memory throughout the execution of the scanner script.

#### Scenario: Capturing logs during scanning
- **WHEN** the daily dependency-check scanner script runs
- **THEN** all log events generated by the logging framework are intercepted and stored in an in-memory buffer in addition to being printed to standard output

### Requirement: MongoDB Archiving
The system SHALL save the captured execution logs as a single document to MongoDB using the collection name configured in the `MONGO_LOGS_COLLECTION` environment variable (defaulting to `execution_logs`) if `MONGO_URI` is defined.

#### Scenario: Archiving logs at completion
- **WHEN** the scanner finishes executing the scanning job and a `MONGO_URI` is configured
- **THEN** it connects to MongoDB and inserts a document containing `job_id`, `start_time`, `end_time`, `status` (set to `SUCCESS`), and the complete string of concatenated logs

### Requirement: Error Resilience
The system SHALL guarantee that the captured execution logs are archived in MongoDB even if the scanner fails during execution, encounters exceptions, or is terminated prematurely.

#### Scenario: Archiving logs on execution failure
- **WHEN** the scanning script encounters an unhandled exception or initialization failure
- **THEN** it captures the exception details, appends them to the log buffer, inserts the log document into MongoDB with `status` set to `FAILED`, and exits

### Requirement: Collection Name Configuration
The system SHALL allow configuring the target MongoDB collection name via the `MONGO_LOGS_COLLECTION` environment variable.

#### Scenario: Custom collection name configuration
- **WHEN** `MONGO_LOGS_COLLECTION` is set to a custom collection name (e.g., `daily_scanner_logs`)
- **THEN** the execution log document is saved to the `daily_scanner_logs` collection instead of `execution_logs`
